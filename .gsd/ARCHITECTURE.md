## API Contract

Defined by the organizer's test harness (`validate_format.py`):

### `POST /analyze`
- **Request**: `{"prompt": "<user text>"}`
- **Response**: `{"harmful": true/false, "articles": ["Article X", ...]}`
  - If `harmful` is `false`, `articles` must be an empty list `[]`
  - If `harmful` is `true`, `articles` must be a non-empty list of violated articles

### `GET /health`
- **Response**: HTTP 200 when server is ready

## Data Flow

1. Client sends `POST /analyze` with `{"prompt": "..."}`
2. FastAPI validates request via Pydantic schema
3. `analyzer.py` orchestrates the pipeline:
   a. `ccpa_knowledge.py` retrieves relevant CCPA sections
   b. `prompt_builder.py` constructs LLM prompt with context + user query
   c. `llm_engine.py` runs inference on local HuggingFace model
   d. `response_parser.py` extracts `harmful` flag and `articles` list
4. Response returned as `{"harmful": bool, "articles": [...]}`
